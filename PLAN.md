CHECKS ДЛЯ API GATEWAY

1. AuthCheck
   Описание:
   Проверяет аутентификацию через внешний сервис.
   Как работает:

* берёт данные из запроса (обычно headers)
* делает HTTP запрос во внешний auth сервис
* проверяет статус ответа
* извлекает данные из ответа (body/headers)
* сохраняет их в context

---

2. HeaderRequiredCheck
   Описание:
   Проверяет наличие обязательных headers в запросе.
   Как работает:

* проходит по списку headers
* если хотя бы один отсутствует → ошибка

---

3. QueryRequiredCheck
   Описание:
   Проверяет наличие обязательных query параметров.
   Как работает:

* читает query из URL
* проверяет наличие нужных ключей
* если нет → ошибка

---

4. IPWhitelistCheck
   Описание:
   Разрешает доступ только с определённых IP.
   Как работает:

* получает IP клиента (RemoteAddr / X-Forwarded-For)
* проверяет, есть ли он в whitelist
* если нет → ошибка

---

5. RateLimitCheck (базовый)
   Описание:
   Ограничивает количество запросов.
   Как работает:

* хранит счётчик запросов (например, по IP)
* если превышен лимит → ошибка
* иначе увеличивает счётчик

---

6. InjectCheck (или StaticValuesCheck)
   Описание:
   Добавляет фиксированные значения в context.
   Как работает:

* берёт значения из config
* кладёт их в context
* всегда проходит

---

7. TimeoutCheck
   Описание:
   Ограничивает время выполнения запроса.
   Как работает:

* создаёт context с timeout
* если downstream не успел → отмена

---

8. MethodCheck (опционально)
   Описание:
   Проверяет HTTP метод запроса.
   Как работает:

* сравнивает request.Method с разрешённым
* если не совпадает → ошибка

---

9. ExternalPolicyCheck (расширение auth)
   Описание:
   Проверяет запрос через внешний policy сервис.
   Как работает:

* отправляет данные запроса во внешний сервис
* получает решение (allow/deny)
* при allow может обогатить context

---

ПРИНЦИПЫ CHECKS:

* каждый check делает одну задачу
* check не знает про upstream
* check не изменяет response
* check может:

  * валидировать запрос
  * обогатить context
  * остановить выполнение

Pipeline:
Request → Checks → Context → Upstream
